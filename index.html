jqcached

Sexy caching for your DOM selectors

jqcached is a jQuery plugin that makes DOM selector caching extremely easy.

Make your application perform better by querying the DOM once and using the selectors without any additional overhead of re-querying them ever again.

Why use it?

  1. The DOM is slow, it is a bottleneck for each JavaScript application. Respect it - cache and reuse.
  2. In case you change the name of element's class or ID, you have to change it only at one place instead of all over the code.
  3. It creates convenient coding convention.

Download it from this public Git repository. Read about it on my blog. And come say 'hi' on Twitter.

How-to


// Cache by assigning to the $.ss object
$.jqcached({
	'navMenu': $('#navMenu'),
	'home': $('#navMenu').find('li:first')
	'nextPage': $('a.nextPage')
});


// Use
$.ss.navMenu.show();
$.ss.home.bind('click', function () {
	alert("We are in bee's knees");
});
$.ss.nextPage.addClass('highlight');

How-to with context


// Or cache by assigning to a passed object
var myMenuView = {};
$.jqcached({
	'navMenu': $('#navMenu'),
	'home': $('#navMenu').find('li:first')
	'nextPage': $('a.nextPage')
}, myMenuView);


// Use
myMenuView.ss.navMenu.show();
myMenuView.ss.home.bind('click', function () {
	alert("We are in bee's knees");
});
myMenuView.ss.nextPage.addClass('highlight');
